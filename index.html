PROJET PHARE • STAGE 2025

Cartographie Scientifique

Contexte & Objectifs

Stage réalisé à l'IRIT (Institut de Recherche en Informatique de Toulouse). L'objectif était de concevoir une plateforme complète de pilotage de la production scientifique du laboratoire, permettant aux décideurs de visualiser et analyser les publications, collaborations et tendances de recherche.

Architecture Technique

  • API REST Flask sécurisée avec authentification JWT
  • Modélisation Oracle avec requêtes récursives complexes (CTE)
  • Dashboards interactifs Chart.js pour la restitution visuelle
  • Architecture MVC modulaire et documentée

Défis Techniques

La complexité résidait dans la modélisation des hiérarchies organisationnelles via des requêtes récursives Oracle et dans la sécurisation de l'API avec un système de tokens JWT. L'optimisation des requêtes sur un volume important de données scientifiques a été essentielle.

Résultats & Apprentissages

  • Plateforme fonctionnelle déployée et utilisée par l'équipe IRIT
  • Maîtrise approfondie de Flask, Oracle et de la sécurité API
  • Rédaction d'une documentation technique complète
  • Compétences en gestion de projet autonome
Python / Flask Oracle SQL JWT Auth Chart.js API REST
ANALYSE & ALGORITHMIQUE

BingeMarking Engine

Contexte & Objectifs

Conception d'un moteur de recommandation intelligent capable de suggérer des séries TV similaires en se basant sur l'analyse textuelle des descriptions. Le projet exploite un corpus de 128 séries pour calculer des scores de similarité via l'algorithme TF-IDF.

Pipeline Technique

  • Scraping et nettoyage du corpus de données (128 séries)
  • Tokenisation et vectorisation TF-IDF avec scikit-learn
  • Calcul de similarité cosinus entre les vecteurs
  • Interface Flask pour l'interaction utilisateur

Défis Techniques

L'enjeu principal était d'optimiser la qualité des recommandations en affinant le prétraitement textuel (suppression des stop-words, lemmatisation) et en trouvant les bons paramètres TF-IDF pour maximiser la pertinence des résultats.

Résultats & Apprentissages

  • Moteur fonctionnel avec recommandations pertinentes
  • Maîtrise du NLP basique et de la vectorisation textuelle
  • Compréhension approfondie des algorithmes de similarité
  • Pipeline de données complet de bout en bout
Python Flask scikit-learn TF-IDF NLP
CONCEPTION DÉCISIONNELLE

Entrepôt de Données

Contexte & Objectifs

Mise en place d'une infrastructure décisionnelle complète pour centraliser et historiser les données d'une organisation. Le projet couvre l'intégralité du cycle décisionnel : extraction, transformation, chargement (ETL) et restitution via tableaux de bord.

Architecture Technique

  • Modélisation dimensionnelle en étoile (Faits / Dimensions)
  • Flux ETL complets avec Talend Open Studio
  • Base de données Oracle pour le stockage
  • Historisation de type SCD (Slowly Changing Dimensions)

Défis Techniques

La gestion de l'historisation des données (type 2 SCD) et la conception de flux ETL robustes capables de gérer les cas d'erreur et les mises à jour incrémentales ont constitué les principaux défis techniques de ce projet.

Résultats & Apprentissages

  • Entrepôt fonctionnel avec données historisées
  • Maîtrise de Talend Open Studio et des flux ETL
  • Compréhension de la modélisation dimensionnelle
  • Capacité à concevoir une architecture BI complète
Talend ETL Oracle SQL Modélisation SCD Type 2
DÉVELOPPEMENT SERVEUR

Gestion d'Effectifs API

Contexte & Objectifs

Conception d'un service REST robuste pour la gestion d'effectifs sportifs. Le projet met l'accent sur une architecture modulaire, la sécurisation des endpoints via JWT et la documentation API avec Postman.

Architecture Technique

  • API REST avec endpoints CRUD complets
  • Authentification et autorisation via JWT
  • Architecture modulaire (routes, controllers, models)
  • Tests et documentation Postman

Défis Techniques

L'implémentation d'un système d'authentification sécurisé avec gestion des rôles et des permissions, ainsi que la structuration modulaire du code pour garantir maintenabilité et scalabilité ont été les principaux défis.

Résultats & Apprentissages

  • API fonctionnelle et entièrement documentée
  • Maîtrise des patterns d'authentification JWT
  • Compétences en architecture logicielle modulaire
  • Collaboration Git en équipe avec workflow branching
Java Python JWT Git Postman
